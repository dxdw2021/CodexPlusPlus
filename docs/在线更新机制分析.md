| 步骤 | 请求目标 | 用途 |

| 2 | 安装包 `.exe` / `.dmg` | 针对当前平台选中的安装包，下载到本地后启动安装器 |

## 3. 核心流程详解

### 3.1 `check_for_update()` — 检查更新

```
check_for_update(current_version)
  │
  ├─ HTTP GET → DEFAULT_LATEST_JSON_URL
  │  URL: https://github.com/BigPizzaV3/CodexPlusPlus/releases/latest/download/latest.json
  │  Header: Accept: application/json
  │  User-Agent: Codex++/X.X.X
  │
  ├─ 解析 JSON 得到所有平台的安装包列表
  │
  ├─ select_update_asset() → 按当前 OS + CPU 架构排名，选最佳安装包
  │
  └─ is_newer_version(release.version, current_version) → 语义版本比较
      返回 UpdateCheck { update_available, latest_version, asset_name, asset_url, ... }
```

**版本比较逻辑** (`parse_version_tag`)：

```
"v1.2.17"      → [1, 2, 17]
"v1.2.17-beta" → [1, 2, 17]   ← 后缀被截断，只比较纯数字段
"1.2.3"        → [1, 2, 3]
```

逐段比较，短的一方补 0，严格大于才算新版本。

### 3.2 `perform_update()` — 下载并安装

```
perform_update(release, download_dir)
  │  download_dir = app_state_dir / "updates"
  │
  ├─ HTTP GET → release.asset_url
  │  （从 latest.json 中解析出的平台专属安装包地址）
  │
  ├─ safe_asset_name() → 防路径穿越校验
  │   √ "CodexPlusPlus-1.2.17-windows-x64-setup.exe" → OK
  │   × "../malware.exe" → bail!
  │   × "/etc/passwd"    → bail!
  │
  ├─ std::fs::write(download_dir / safe_name, bytes)
  │
  └─ launch_installer(path)
       Windows: CreateProcess (CREATE_NO_WINDOW) → 无窗口启动安装向导
       macOS:   `open xxx.dmg` → 挂载 DMG，用户手动拖入 Applications
```

## 4. 安装包选择算法

### 4.1 排名规则

`platform_asset_rank(name)` 返回 0/1/2，数值越低越优先：

| 平台 | rank 0（精确匹配） | rank 1（降级兼容） | rank 2（不匹配） |
|---|---|---|---|
| Windows | `codex` + `plus` + `.exe`（不区分 x64/arm64） | — | 其他所有 |
| macOS x64 | `codex` + `plus` + `.dmg` + 含 `x64` | `codex` + `plus` + `.dmg` + 含 `arm64` | 无 `.dmg` |
| macOS arm64 | `codex` + `plus` + `.dmg` + 含 `arm64` | `codex` + `plus` + `.dmg` + 含 `x64` | 无 `.dmg` |
| Linux | — | — | 全 rank 2 → `None` |

### 4.2 选择伪代码

```python
def select_update_asset(assets):
    best = None
    best_rank = ∞
    for name, url in assets:
        rank = platform_asset_rank(name.lower())
        if rank >= 2:    # 跳过不支持的平台
            continue
        if rank < best_rank:  # 数值越小越优先
            best = (name, url)
            best_rank = rank
    return best  # 可能为 None
```

### 4.3 macOS 架构区分

文件名匹配两种格式（兼容新旧命名风格）：

```
新格式: CodexPlusPlus-1.2.17-macos-x64.dmg     ← "-x64." 或 "-arm64."
旧格式: CodexPlusPlus_1.0.9_x64.dmg            ← "_x64." 或 "_arm64."
```

### 5.1 触发条件


### 5.2 构建 Job

| Job | 运行环境 | 构建产物 |
|---|---|---|
| `windows-installer` | `windows-latest` | `CodexPlusPlus-{version}-windows-x64-setup.exe` |
| `windows-installer` | `windows-latest` | `CodexPlusPlus-{version}-windows-x64.zip` |
| `macos-dmg` (x64) | `macos-15-intel` | `CodexPlusPlus-{version}-macos-x64.dmg` |
| `macos-dmg` (x64) | `macos-15-intel` | `CodexPlusPlus-{version}-macos-x64.zip` |
| `macos-dmg` (arm64) | `macos-14` | `CodexPlusPlus-{version}-macos-arm64.dmg` |
| `macos-dmg` (arm64) | `macos-14` | `CodexPlusPlus-{version}-macos-arm64.zip` |

### 5.3 `latest.json` 生成逻辑

所有构建 Job 完成后，`latest-json` Job 执行：

```javascript
const baseUrl = `https://github.com/${repo}/releases/download/${tag}`;
const assets = release.assets
  .filter(name => name !== "latest.json")
  .map(name => ({ name, url: `${baseUrl}/${encodeURIComponent(name)}` }));
```

然后通过 `gh release upload` 上传到当前 Release 的 assets 中，**覆盖**已有的 `latest.json`。

```typescript
  const result = await call("check_update");

// 执行更新
const performUpdate = async () => {
  // 1. 伪进度条：每 500ms +10%，最大 92%
  const progressTimer = setInterval(() => {
    setUpdateInstallProgress(prev => {
      if (!prev.active) return prev;
      return { ...prev, percent: Math.min(92, prev.percent + 10) };
    });
  }, 500);

  // 2. 调用 Tauri 后端下载 + 启动安装器
  const result = await call("perform_update", { release });

  // 3. 清理定时器，进度设为 100%
  clearInterval(progressTimer);
  setUpdateInstallProgress({ active: false, percent: 100, ... });
};
```

## 8. 移植到非开源项目

### 8.1 原理分析

当前更新机制**完全不依赖 GitHub API**，仅使用纯 HTTP GET 下载文件：

```rust
// update.rs → fetch_latest_release() → 核心就一行 HTTP GET
pub fn proxied_client(user_agent: &str) -> anyhow::Result<reqwest::Client> {
    Ok(reqwest::Client::builder().user_agent(ua).build()?)  // ← 没有 token / OAuth
}
```

### 8.2 需要改什么

**最低改动**——只需替换一个常量：

```rust
// 改前：指向 GitHub Releases
pub const DEFAULT_LATEST_JSON_URL: &str =
    "https://github.com/BigPizzaV3/CodexPlusPlus/releases/latest/download/latest.json";

// 改后：指向你自己的更新服务器
pub const DEFAULT_LATEST_JSON_URL: &str =
    "https://your-cdn.com/my-app/latest.json";
```

### 8.3 适配场景

| 场景 | 需要改动 | 工作量 |
|---|---|---|
| 文件放公共 CDN / S3 / 自建 Nginx | 改 URL 常量 | 1 行 |
| 需要 Basic Auth 访问 | `proxied_client()` 加 `.default_headers()` | 3 行 |
| 需要 Bearer Token 访问 | `proxied_client()` 加 `bearer_auth()` | 3 行 |
| 文件放 GitHub **私有**仓库 Releases | 需加 `Authorization: Bearer <GITHUB_TOKEN>` | 3-5 行 |
| 自定义版本号格式 | 改 `parse_version_tag()` 的解析逻辑 | 按需 |
| 自定义安装包命名规则 | 改 `is_windows_installer_asset()` / `is_macos_installer_asset()` | 按需 |

### 8.4 推荐私有源方案

```
你的更新服务器
├── latest.json              ← 版本清单（手动更新或 CI 生成）
├── v1.2.17/
│   ├── MyApp-windows-x64-setup.exe
│   └── MyApp-macos-arm64.dmg
└── v1.2.18/
    ├── MyApp-windows-x64-setup.exe
    └── MyApp-macos-arm64.dmg
```

`latest.json` 内容：

```json
{
  "version": "v1.2.18",
  "url": "https://your-cdn.com/releases/v1.2.18",
  "body": "新版本发布说明...",
  "assets": [
    {
      "name": "MyApp-windows-x64-setup.exe",
      "url": "https://your-cdn.com/releases/v1.2.18/MyApp-windows-x64-setup.exe"
    },
    {
      "name": "MyApp-macos-arm64.dmg",
      "url": "https://your-cdn.com/releases/v1.2.18/MyApp-macos-arm64.dmg"
    },
    {
      "name": "MyApp-macos-x64.dmg",
      "url": "https://your-cdn.com/releases/v1.2.18/MyApp-macos-x64.dmg"
    }
  ]
}
```
